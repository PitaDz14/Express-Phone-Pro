
# Express Phone Pro | إكسبريس فون برو 📱

نظام متكامل واحترافي لإدارة محلات الهواتف النقالة، مصمم ليعمل بكفاءة عالية على الهواتف والحواسيب مع دعم كامل للعمل بدون إنترنت (Offline-First).

## 🚀 الميزات الرئيسية

*   **نظام نقطة البيع (POS) الذكي:**
    *   دعم كامل للباركود وكود QR عبر كاميرا الهاتف.
    *   معاينة حرارية (Thermal Preview) احترافية قبل الطباعة.
    *   التحقق التلقائي من المخزون ومنع بيع كميات غير متوفرة.
*   **إدارة المخزون المتقدمة:**
    *   فرز (Filtering) متعدد التصنيفات في آن واحد.
    *   نظام تصنيفات هرمي (مثل: قطع غيار > شاشات > سامسونج).
    *   تنبيهات ذكية للمخزون المنخفض مع ترتيب تفاعلي وتصدير بصيغ (CSV, TXT).
*   **نظام الصلاحيات (RBAC):**
    *   **المدير (Admin):** صلاحيات كاملة، إحصائيات مالية، إدارة أسعار الشراء، وإدارة الطاقم.
    *   **الموظف (Worker):** عمليات البيع والديون فقط، مع إخفاء كامل لأسعار الشراء والإحصائيات الحساسة.
*   **الديون والمالية:**
    *   تتبع دقيق لديون الزبائن مع سجل تاريخي لكل فاتورة.
    *   زر "إعادة المزامنة" لتصحيح الحسابات تلقائياً بناءً على سجل الفواتير.
    *   إحصائيات حصرية لقطاع الشاشات (قيمة الشراء والبيع).
*   **الأمان والتقنية:**
    *   نظام تتبع العمليات (Audit Logs) لمعرفة من قام بكل عملية (مسجل باسم الموظف).
    *   قاعدة بيانات Firebase مع مزامنة لحظية ودعم العمل أوفلاين.
    *   واجهة مستخدم متجاوبة تماماً تميز المستخدم النشط بالألوان.

## ⚙️ دليل الانتقال لمشروع Firebase جديد

إذا أردت تغيير حساب Firebase أو المشروع، اتبع الخطوات التالية:

1.  **Firebase Console:**
    *   أنشئ مشروعاً جديداً وفعل **Firestore** و **Authentication** (البريد وكلمة المرور).
    *   أضف Web App وانسخ كائن `firebaseConfig`.
2.  **تحديث الإعدادات:**
    *   استبدل القيم في ملف `src/firebase/config.ts` بالقيم الجديدة.
3.  **قواعد الأمان:**
    *   انسخ محتوى ملف `firestore.rules` من المشروع الحالي والصقه في قسم **Rules** في Firestore بمشروعك الجديد.
4.  **المستخدم الأول:**
    *   أنشئ مستخدماً في المشروع الجديد وانسخ الـ **UID** الخاص به.
    *   أضف وثيقة في مجموعة `user_roles` يكون معرّفها هو الـ UID، وتحتوي على الحقول: `role: "Admin"`, `username: "المدير"`.

## 📦 حل مشكلة الرفع (403 Forbidden)
إذا واجهت خطأ `403` عند محاولة الـ `push` إلى GitHub:

1.  **إعدادات الـ Token:**
    *   تأكد من استخدام **Fine-grained token**.
    *   امنح صلاحية **Read and Write** لخاصية **Contents** لهذا المستودع تحديداً.
2.  **أوامر الـ Terminal:**
    ```bash
    git remote remove origin
    git remote add origin https://YOUR_TOKEN_HERE@github.com/PitaDz14/Express-Phone-Pro.git
    git push -u origin main
    ```
